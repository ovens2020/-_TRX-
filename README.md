批量生成解决什么问题
单个地址生成很快，但靓号命中具有概率性。批量生成可以持续输出候选地址，并按用户规则筛出少量高价值结果。

在TRX和TRC20收款场景中，批量筛选可以同时覆盖不同后缀、顺子、多连和品牌字符组合。

Batch#001: T...8888
Batch#002: T...TRX...9999
多线程与GPU并行
批量生成把任务拆分成大量独立候选钱包，线程或GPU核心负责并行扫描，再把命中结果汇总。

用户可以根据设备性能调整强度，平衡速度、温度和系统响应。

批量结果如何安全保存
批量导出时尤其要注意文件加密和存储位置。包含私钥的结果文件不应放入网盘同步目录。

建议将最终选中的钱包单独备份，未使用的候选私钥及时清理，减少管理负担。

批量生成流程
设置批量规则，例如后缀8888和四连保留。
运行GPU生成器并观察匹配数量。
导出命中结果并筛选最适合展示的地址。
逐个导入验证，保留最终使用的钱包。
